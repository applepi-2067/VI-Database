MXPIOExpansion  
One-sentence summary of this page.
Updated Dec 29, 2014 by scottlib...@gmail.com

Introduction

The navX MXP breaks out all usable signal pins on the RoboRIO / MyRIO MXP Connector.

DC Voltage Selection

Using the provided jumper, select the DC Voltage which will be routed to each of the connector blocks. Select from either 3.3V or 5V DC. This regulated voltage is supplied directly by the host computer (e.g., the RoboRIO).

Digital I/Os

Each of the 10 Digital I/O pins on the MXP connector has a corresponding 3-pin connector (DC Voltage, Ground and Signal). On the RoboRIO, these may be configured for use as either GPIO Inputs or Outputs, PWM Inputs or Outputs, or Quadrature Encoder Inputs.

Analog Inputs

Each of the 4 Analog Input pins on the MXP connector has a corresponding 3-pin connector (DC Voltage, Ground and Signal). On the RoboRIO, these signals are routed to the internal Analog-to-Digital Converters (ADCs).

Analog Outputs

Each of the 2 Analog Output pins on the MXP connector has a corresponding 3-pin connector (DC Voltage, Ground and Signal). On the RoboRIO, these signals are generated by the host computer's internal Digital-to-Analog Converters (DACs).

I2C

The navX MXP I2C connector can be used to connect the RoboRIO to an external I2C Device. The RoboRIO functions as an I2C Master. The connector provides DC Voltage, Ground, Clock (SCL) and Data (SDA).

Note that this I2C connector resides on the same I2C bus which may optionally be used to communicate between the RoboRIO and the navX MXP's onboard processor. The navX MXP uses I2C Address 50 (0x32), so be sure that any external I2C device does not use this address.

SPI

The navX MXP SPI connector can be used to connect the RoboRIO to an external SPI device. The RoboRIO functions as an SPI Master. The connector provides DC Votage, Ground, Clock (SCK), Slave Select (SS), Master-in/Slave-out (MISO) and Master-out/Slave-in (MOSI) signals.

Note that this SPI connector resides on the same SPI bus which may optionally be used to communicate between the RoboRio and the navX MXP's onboard processor. The navX MXP will respond to the Slave Select signal if and only if the SPI Enable dip switch is set to the "ON" position. Thus, the SPI Enable dip switch should be set to the "OFF" position if you wish to communicate with an external device via the SPI connector.

TTL UART

The navX MXP TTL UART connector can be used to connect the RoboRIO to an external TTL-level UART device.

NOTE: The TTL UART connector cannot be used to connect to an external RS-232 signal, since RS-232 voltages are much higher than TTL-level UART voltages. Connecting a higher-voltage RS-232 device to the TTL UART connector may subject the RoboRIO to damaging voltage levels on these pins.

Note that this TTL UART connector can be used to communicate between the RoboRIO and the navX MXP's onboard processor (in fact, this is the default). The navX MXP will respond to the UART TX signal from the RoboRIO if and only if the UART Enable dip switch is set to the "ON" position. Thus, the UART Enable dip switch should be set to the "OFF" position if you wish to communicate with an external device via the TTL UART connector.
